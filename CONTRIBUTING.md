# Contributing Guide

Thank you for helping improve this interview question bank. Keep contributions original, practical, and interview-ready.

## Core rules

- Use the exact question template defined in `QUESTION_BANK.md`.
- Keep questions and guidance in English.
- Focus on realistic engineering interviews, not textbook definitions.
- **Do not paste or lightly paraphrase copyrighted material. All contributions must be written in your own words.**
- Do not submit content generated by copying another repository's structure.

## Local validation before opening a PR

1. Run structure validation:
   ```bash
   python scripts/validate_bank.py
   ```
2. Run markdown linting (if available locally):
   ```bash
   markdownlint "**/*.md"
   ```
3. Confirm that `README.md` content is in sync with `QUESTION_BANK.md` and introduction text.

## Pull request checklist (required)

In your PR description, confirm all items below:

- [ ] I used the required question template exactly.
- [ ] I wrote all new material in my own words.
- [ ] I did not paste or lightly paraphrase copyrighted material.
- [ ] I ran `python scripts/validate_bank.py` locally.
- [ ] I verified the README sync behavior remains correct.
